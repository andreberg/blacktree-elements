Skype API for OSX
By mat.taylor. Posted 1 week, 6 days ago in Documentation.

Introduction

  • Download the latest Skype for Mac beta. API is supported in version 1.0.0.46 or greater - so the current non beta version is not supported at the moment.
  • Download Skype.framework.
  • Review the Skype API protocol documentation.

Skype API for Mac has currently Cocoa, Carbon and AppleScript interfaces. Cocoa and Carbon interfaces are implemented in Skype.framework. The recommended way to use the Skype framework is to include it in your application as an embedded framework. Copy it into your application bundle and link it to your application.

Client applications can control Skype by sending string commands. The format of these strings commands is described in Skype API protocol documentation. If you are using Cocoa or Carbon interface, then Skype will send information back to your application by calling asynchronous delegate functions/methods (see below).

Cocoa

Skype Delegate Object

Required Method

Design an object in your application to be a Skype delegate. This object must implement the following method: 

   @protocol SkypeAPIDelegate
      (NSString*)clientApplicationName;
   @end

This method should return the name of your application. This name will be shown to the user, when your application uses Skype. The name should not include any version information.

Optional methods

In order to receive information from Skype, it is recommended that you implement the following methods in your application: 

    ( void ) skypeNotificationReceived:(NSString*)aNotificationString;

This is the main delegate method Skype uses to send information to your application. aNotificationString is a Skype API string as described in Skype API protocol documentation. 

   (void) skypeAttachResponse:(unsigned)aAttachResponseCode;

This method is called after Skype API client application has called [SkypeAPI connect];. aAttachResponseCode is 0 on failure and 1 on success. 

   (void) skypeBecameAvailable:(NSNotification*)aNotification;

This method is called after Skype has been launched.

   (void) skypeBecameUnavailable:(NSNotification*)aNotification;

This method is called after Skype has quit.

Guidelines

The first method your application should call is 

   (void) setSkypeDelegate:(NSObject*)aDelegate,

where aDelegate is your Skype delegate object. In most implementations, that will probably be: 

   [SkypeAPI setSkypeDelegate:self];

Next, you should call 

   [SkypeAPI connect];

After you have received positive response with 

   (void) skypeAttachResponse:(unsigned)aAttachResponseCode,

you can start sending commands to Skype by using + 

   (void) sendSkypeCommand:(NSString*)aCommandString,

where aCommandString is command string as described in Skype API protocol documentation. For example: 

   [SkypeAPI sendSkypeCommand:@"CALL echo123"];

When your application quits or wants to disconnect from Skype, you should call 

   [SkypeAPI disconnect];

Carbon

Skype Delegate Struct

In order to use Skype API, you must create a single instance of struct SkypeDelegate . If you set callback functions for the members of this struct, then Skype will call these functions to send information to your application. The only required member of this struct is a string clientApplicationName . This name will be shown to the user, when your application uses Skype. The name should not include version information.

Here is the definition of SkypeDelegate : 

   struct SkypeDelegate {
      // Required member
      CFStringRef clientApplicationName;
      // Optional members, can be NULL
      void (*SkypeNotificationReceived) (CFStringRef aNotificationString);
      void (*SkypeAttachResponse)(unsigned int aAttachResponseCode);
      void (*SkypeBecameAvailable)(CFPropertyListRef aNotification);
      void (*SkypeBecameUnavailable)(CFPropertyListRef aNotification);
   };

In order to receive information from Skype, it is recommended that you implement the following callback functions and set them in the SkypeDelegate struct: 

   void (*SkypeNotificationReceived)(CFStringRef aNotificationString);

This is the main delegate method Skype uses to send information to your application. aNotificationString is a Skype API string as described in Skype API protocol documentation. 

   void (*SkypeAttachResponse)(unsigned int aAttachResponseCode);

This method is called after Skype API client application has called 

   ConnectToSkype()

aAttachResponseCode is 0 on failure and 1 on success. 

   void (*SkypeBecameAvailable)(CFPropertyListRef aNotification);

This method is called after Skype has been launched. 

   void (*SkypeBecameUnavailable)(CFPropertyListRef aNotification);

This method is called after Skype has quit.

You should define the functions like this: 

   void SkypeNotificationReceived(CFStringRef aNotificationString){}
   void SkypeAttachResponse(unsigned int aAttachResponseCode){}
   void SkypeBecameAvailable(CFPropertyListRef aNotification){}
   void SkypeBecameUnavailable(CFPropertyListRef aNotification){}

and you can set them as members of your SkypeDelegate struct like so: 

   SkypeDelegate mySkypeDelegate;
   mySkypeDelegate.SkypeNotificationReceived = SkypeNotificationReceived;
   mySkypeDelegate.SkypeAttachResponse = SkypeAttachResponse;
   mySkypeDelegate.SkypeBecameAvailable = SkypeBecameAvailable;
   mySkypeDelegate.SkypeBecameUnavailable = SkypeBecameUnavailable;
   mySkypeDelegate.clientApplicationName = CFSTR("My Carbon App");

Guidelines

The first method your application should call is 

   void SetSkypeDelegate(struct SkypeDelegate* aDelegate),

where aDelegate is your SkypeDelegate struct. In most implementations, that will probably be: 

   SetSkypeDelegate(&myCarbonDelegate);

Next, you should call 

   ConnectToSkype();

After you have received positive response with 

   void (*SkypeAttachResponse)(unsigned int aAttachResponseCode),

you can start sending commands to Skype by using 

   void SendSkypeCommand(CFStringRef aCommandString),

where aCommandString is command string as described in Skype API protocol documentation. For example: 

   SendSkypeCommand(CFSTR("CALL echo123"));

When your application quits or wants to disconnect from Skype, you should 

   call DisconnectFromSkype();

AppleScript

There is just one command for Skype API, but it is a very powerful command, because you can send most of the command strings as specified in Skype API protocol documentation.

Examples: 

   tell application "Skype"
      send command "MESSAGE echo123 check" script name "My Script"
   end tell

   tell application "Skype"
      send command "CALL echo123" script name "My Other Script"
   end tell
